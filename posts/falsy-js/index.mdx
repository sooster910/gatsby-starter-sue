다시 짚어보는 null, undefined, falsy

다음과 같은 value들은 falsy로 간주된다.

1. false
2. 0 (zero)
3. -0 (minus zero)
4. 0n (BigInt zero)
5. '', "", `` (empty string)
6. null
7. undefined
8. NaN

다음과 같은 value들은 truthy로 간주된다.

1. '0' (a string containing a single zero)
2. 'false' (a string containing the text “false”)
3. [] (an empty array)
4. {} (an empty object)
5. function(){} (an “empty” function)

```javascript
if (value) {
  // value is truthy
} else {
  // value is falsy
  // it could be false, 0, '', null, undefined or NaN
}
```

`a` 는 비어있는 **객체** truthy 이므로 type이 undefined가 될 수 없다.

```javascript
const a = {}
console.log(typeof a === 'undefined') //false
```

Null값과 undefined 값 체크

```javascript
const isNullUndefined = (arg) => {
  return arg == null
}
console.log(variable == null)
```

객체와 배열은 항상 truthy 이므로 null과 undefined를 구별해줄 필요가 없다.

##### 원시값들에 대한 null 또는 undefined 체크, empty string 포함

```javascript
const isNullorUndefined =(val) =>(typeof a !== "undefined") && a? false: true
)

isNullorUndefined([]) //always return false


```

배열또는 객체를 이용하면 항상 false가 리턴된다. 그래서 빈객체, 빈 배열을 구분할 수가 없다.

객체 또는 배열의 null, undefined를 구별하고 싶다면,

```javascript
if (typeof value !== 'undefined' && value) {
  //여기엔 항상 true
}
```
